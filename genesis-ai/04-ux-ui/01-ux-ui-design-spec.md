## 一、设计原则 (Design Principles)

### 1.1 核心原则

基于 2025-2026 年流行的 DeFi 平台设计趋势（Uniswap、RockFlow、1inch）：

| 原则 | 说明 | 参考案例 |
|:--|:--|:--|
| **简洁优先** | 去除冗余，核心操作一步到达 | Uniswap |
| **卡片式布局** | 信息模块化，层次分明 | RockFlow |
| **深色默认** | 专业、护眼、加密货币行业标准 | PancakeSwap |
| **移动端优先** | 响应式设计，覆盖所有设备 | 1inch |
| **实时数据** | WebSocket 推送，价格即时更新 | Binance |
| **渐进式披露** | 新手模式 + 高级模式切换 | Uniswap |

## 二、设计系统 (Design System)

### 2.1 技术选型

| 层级 | 选型 | 理由 |
|:--|:--|:--|
| **UI 框架** | **shadcn/ui** | 2025 年最流行，现代化、可定制 |
| **样式库** | **Tailwind CSS** | 快速开发，响应式友好 |
| **设计工具** | **Figma** | 行业标准，支持 AI 辅助设计 |
| **图标库** | **Lucide React** | shadcn/ui 原生集成，统一风格 |
| **动画库** | **Framer Motion** | 流畅交互，性能优秀 |
| **图表库** | **Recharts** | 轻量级，React 集成 |
| **Web3** | **wagmi.sh + viem** | 最流行的 Web3 钩子库 |

## 三、组件规范 (Component Specifications)

### 3.1 核心组件库

基于 shadcn/ui，定制化以下组件：

| 组件 | 用途 | 定制化 |
|:--|:--|:--|
| **Button** | 主要操作 | 渐变背景、悬停动画 |
| **Card** | 信息展示 | 深色背景、微妙边框 |
| **Input** | 表单输入 | 轮廓式、聚焦高亮 |
| **Select** | 下拉选择 | 箭头动画、清除按钮 |
| **Dialog** | 弹窗 | 背景模糊、动画过渡 |
| **Tabs** | 标签切换 | 下划线动画 |
| **Progress** | 进度条 | 渐变填充、动画 |
| **Badge** | 徽章标签 | 颜色编码（收益/亏损） |
| **Avatar** | 头像 | 默认占位符 |
| **Skeleton** | 加载占位 | 闪烁动画 |
